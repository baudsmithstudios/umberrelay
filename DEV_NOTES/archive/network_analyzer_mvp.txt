MVP: NETWORK BEHAVIOR & PRIVACY ANALYZER (v0.1)

CORE GOAL

Answer one question clearly: “What are the devices on my network doing,
and is anything unusual?”

------------------------------------------------------------------------

SCOPE

IN SCOPE: - DNS-based visibility - Device-level summaries - Basic
anomaly detection - Simple web UI

OUT OF SCOPE: - Packet inspection - TLS decryption - Automatic
blocking - Machine learning models - Deep integrations

------------------------------------------------------------------------

ARCHITECTURE

DATA SOURCE: - Pi-hole API/logs (polled every 10–30 seconds)

WHY: - Widely used - Structured DNS logs - No packet capture required

BACKEND: - Language: Go or Node.js - Database: SQLite (zero-config,
lightweight)

DATA MODEL:

devices: - id - ip - hostname

dns_queries: - device_id - domain - timestamp

domain_stats: - domain - count - unique_devices

------------------------------------------------------------------------

INGESTION LOOP

-   Poll Pi-hole API every 10–30 seconds
-   Normalize logs into database
-   Aggregate domain statistics

------------------------------------------------------------------------

FEATURES (MVP)

1)  DEVICE OVERVIEW

-   List all devices
-   Show:
    -   total DNS requests
    -   top domains
    -   activity level

WHY: Users think in terms of devices, not raw logs.

2)  TOP TALKERS VIEW

-   Rank devices by:
    -   request volume
    -   number of unique domains

WHY: Quick identification of noisy or suspicious devices.

3)  DOMAIN INSIGHT PANEL

-   For each domain:
    -   list associated devices
    -   show request frequency over time

WHY: Helps determine whether a domain is normal or unusual.

4)  BASIC ANOMALY DETECTION

Rule 1: New Domain - Device contacts a domain it has never seen before

Rule 2: Traffic Spike - Requests exceed 2–3x baseline

Rule 3: New Device - Unknown IP appears on network

WHY: Simple heuristics provide immediate value without complexity.

5)  DAILY SUMMARY (CORE FEATURE)

Example Output: - “2 new devices joined your network” - “Smart TV
contacted 5 new domains” - “No unusual spikes detected”

WHY: Condenses all data into actionable insights. Primary user-facing
value.

6)  BASIC RISK HEURISTICS

-   Tag domains as:
    -   common
    -   unknown
    -   suspicious (via public lists)

WHY: Adds context without deep analysis complexity.

------------------------------------------------------------------------

UI DESIGN

PAGES: 1. Dashboard 2. Devices 3. Domains 4. Alerts

STYLE: - Table-based layout - Minimal frontend framework - Optimized for
Raspberry Pi performance

------------------------------------------------------------------------

USER FLOW

1.  User opens dashboard
2.  Sees summary:
    -   new devices
    -   unusual activity
3.  Clicks into a device
4.  Reviews:
    -   domains contacted
    -   anomalies
5.  Decides whether to investigate further

------------------------------------------------------------------------

MVP TIMELINE

Week 1: - Pi-hole ingestion - SQLite schema - Basic API

Week 2: - Device and domain views - Aggregation logic

Week 3: - Anomaly detection - Summary generation

Week 4: - UI polish - Docker packaging

------------------------------------------------------------------------

KEY DESIGN PRINCIPLES

1)  CLARITY OVER COMPLETENESS

-   Show only what matters

2)  DEVICE-CENTRIC MODEL

-   Everything maps back to a device

3)  CHANGE DETECTION

-   Highlight what is different, not raw logs
